# Skills 滚动条宽度稳定设计

## 目标

Skills 页面展开包含较多成员的 source 时会出现纵向滚动条。滚动条当前占用内容宽度，导致列表右侧向内收缩。展开和收起前后，列表及其操作区应保持相同宽度。

## 方案

- 仅在 Skills 页面的内部滚动容器 `.skills-page` 上设置 `scrollbar-gutter: stable`。
- 保留现有 `overflow: auto`，内容不足时不显示滚动条，但浏览器始终为其预留槽位。
- 不修改共享 `PageLayout` 或 `.main`，避免改变其他页面的可用宽度。
- 窄屏沿用当前页面布局；验证预留槽位不会造成横向溢出。

## 其他页面检查

- MCP 的主要列表和部分详情滚动区已使用 `scrollbar-gutter: stable`，无需重复修改。
- Memory、Variables、通用 SelectableList 及对话框存在内部滚动区，但其布局和 Skills 不同。本次仅检查是否存在同类的整页宽度跳动；没有可复现证据时不扩大改动范围。
- 若检查发现某一页面确有相同宽度跳动，应在该页面实际承担纵向滚动的容器上局部增加稳定槽位，而不是修改共享外壳。

## 验证

- 运行相关 Web 测试。
- 使用 `playwright-cli` 在桌面视口记录展开前后 `.skills-page` 的可见宽度和滚动状态，确认宽度不变且页面可滚动。
- 在窄屏视口检查无横向溢出。
- 浏览其他主要页面，确认没有因本次局部样式产生回归，并记录同类问题分析结果。
