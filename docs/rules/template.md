# 业务规则模板

新增业务规则时，复制本文件中的规则条目模板，并按 [规则索引](index.md) 中的编号前缀选择稳定编号。规则只描述当前产品契约、安全边界和可验证行为，不写历史沿革、内部调用链或实现计划。

## 文件模板

```md
# <领域> 规则

一句话说明本文件覆盖的产品契约范围。

## R-<AREA>-001 <短标题>

Status: active
Applies to: <scope>

Rule:
<必须遵守的产品契约。用当前事实表述，不写“曾经/迁移/计划”。>

Implications:

- <实现、API、UI 或数据表现必须体现的直接后果。>

Safety:

- <不得突破的安全边界，尤其是覆盖、删除、泄密、静默失败、越权访问。>

Examples:

- <一个典型场景，说明规则如何表现。>

Tests:

- <自动化验证入口，例如 packages/<package>/test/<file>.test.ts>
```

## 写作检查

- 一条规则只表达一个稳定契约；如果需要多个“且/或”才能读懂，拆成多条。
- `Rule` 写产品语义，`Implications` 写直接后果，`Safety` 写禁止越界的行为。
- `Examples` 使用当前真实场景，不使用假想未来能力。
- `Tests` 引用已有自动化验证；如果规则尚无测试，应在相关实现改动中补测试。
- 新 feature spec 链接规则编号，不复制规则正文。
